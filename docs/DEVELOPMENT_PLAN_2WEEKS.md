## Текущее состояние

### Что уже реализовано (demo/pilot):
- Авторизация с сессиями, CSRF, ролями (buyer/seller/admin)
- Каталог товаров с фильтрацией (сектор, подкатегория, страна, поиск)
- Карточка товара с полными метаданными
- Система сделок (создание, статусы, документы, чеклисты)
- Мессенджер (потоки, сообщения, системные уведомления)
- Админ-панель (дашборд, управление сделками, пользователи)
- Загрузка файлов (PDF/PNG/JPG/DOCX, валидация по magic bytes)
- Страницы: Landing, Каталог, Аналитика, О платформе, Инвестиции, Логин

### Что НЕ реализовано (по ТЗ):
- Регистрация и верификация пользователей
- Полноценный CRUD продуктов продавца (форма есть, сохранение — нет)
- Каталог инвестиционных проектов (страница-заглушка)
- Контрактный модуль (шаблоны, хранение контрактов)
- Логистический модуль
- Финансово-гарантийный модуль
- Регион происхождения для товаров РК
- Мультиязычность (EN/KZ/RU/CN)
- Профиль компании продавца
- Страница контактов
- Модерация объявлений и верификация в админке

## Приоритеты

- **P0** (Must Have) — без этого платформа не соответствует ТЗ пилотного этапа.
- **P1** (Should Have) — значительно улучшает качество, но можно доработать позже.

## Риски

| Риск | Вероятность | Митигация |
|------|------------|-----------|
| Ошибка сборки Tailwind + Next.js | Высокая | День 10, задача 9.3 — приоритет |
| Объём инвестиционного модуля | Средняя | MVP: каталог + карточка, без полного CRUD |
| In-memory теряет данные при рестарте | Известно | Пилотный этап — допустимо. JSON-файл как fallback |
| Мультиязычность не влезает в 2 недели | Высокая | Вынесено за скоуп — отдельный спринт |

## Что НЕ входит в эти 2 недели

- Мультиязычность (EN/KZ/CN) — отдельный спринт
- Миграция на реальную БД (PostgreSQL)
- Онлайн-платежи и ЭЦП (по ТЗ — не на пилоте)
- KYC/AML интеграция
- Push-уведомления
- PWA / offline-режим
- Нагрузочное тестирование
